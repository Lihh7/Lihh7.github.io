事务

一组操作的集合，不可分割的工作单位。事务会把所有操作作为一个整体一起向系统提交或者撤销操作，要么全部成功，要么全部失败。

默认MySQL的事务自动提交的，执行一条DML语句，会隐式提交事务。

操作

  • 开启事务 START TRANSACTION 或者 BEGIN

  • 提交 COMMIT

  • 回滚 ROLLBACK

四大特性 ACID

  • 原子性：不可分割

  • 一致性：所有数据保持一致状态

  • 隔离性：不受外部并发操作影响的独立环境运行

  • 持久性：改变永久

并发事务问题

  • 脏读：读到未提交的数据

  • 不可重复读：先后读取同一条数据，但是读取的不同

  • 幻读：查询时没有对应数据行，插入的时候又存在

事务的隔离级别

  • Read uncommitted：脏读，不可重复读，幻读

  • Read committed：不可重复读，幻读

  • Repeatable Read：幻读

  • Serializable（串行化）
