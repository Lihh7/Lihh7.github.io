Linux 基本用法

  • 服务器领域，高效免费

  • 嵌入式领域，可以裁剪使得内核小

发行版是对它进行包装，比如 Ubuntu，RedHat，CentOS

一般安装网络选择 nat 模式，不容易造成 ip 冲突。设置后使用 ISO 映射文件

网络连接的三种模式

  • 桥接模式：主机假如是 192.168.0.20 虚拟系统可能会是 192.168.0.80 容易造成 ip 冲突

  • nat 模式：网络地址转换，主机会生成虚拟网卡，内部形成独立的网段，通过主机代理和外网通讯，不造成 ip 冲突

  • 主机模式：独立系统

linux 文件系统采用级层式的树状目录结构，有一个根目录 / ，然后分出其他目录（已经规划好）。它会把硬件映射成文件形式（一切皆为文件）

具体的目录结构

  • /bin 即 Binary 存放常用的命令

  • /sbin 其中的 s 是 Super User ，放着系统管理员使用的系统管理程序

  • /home 存放普通用户的主目录，每个用户有一个自己的目录，一般以用户账号名命名

  • /root 为系统管理员，超级权限者的用户主目录

  • /lib 系统开机所需要的最基本的动态连接共享库，类似于 Windows 的 DLL 文件。几乎所有程序都需要这些共享库

  • /lost+found 一般情况为空，如果非法关机存放了一些文件

  • /etc 所有的系统管理所需要的配置文件和子目录，比如安装 mysql 数据库 my.conf

  • /usr 用户的程序和文件放在这个目录，类似于 program files

  • /boot 放的是启动相关文件

  • /proc 虚拟目录，系统内存的映射，访问这个目录来获取系统信息（不能动）

  • /srv service 存放服务启动后需要提取的数据，不能动

  • /tmp 临时存放文件

  • /dev device类似于设备管理器，把所有硬件映射成文件形式

  • /media 自动识别一些设备例如u盘和光驱，识别后挂载到这个目录下

  • /mnt 给用户临时挂载别的文件系统，可以将外部的存储挂载到 /mnt/ 上

  • /opt 主机额外安装软件所存放的目录，一般先把安装文件放在这里

  • /usr/local 另一个给主机额外安装软件的目录，一般是通过编译源码安装的程序

  • /var 存放着不断扩充的东西，经常改动的比如日志

  • selinux security-enhanced linux 是一个安全子系统，能控制程序只访问特定文件，三种工作模式可以设置
